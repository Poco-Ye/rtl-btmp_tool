    MP tool 文档补充说明

    Skip to end of metadata
    • Created by 居文, last modified on 2016/06/28
    Go to start of metadata
    1.   当前蓝牙接收测试是否有MAC过滤功能，如果有该如何增加?
    2.    bt_mp_SetParam 1,0;2,0;3,7;4,0;6,0x7F;7,0x7;11,0x000000c6967e  红色字体部分MP tool里面默认为none，而在案例里面却有数值，这个部分具体有什么用？
    RTK==》（1和2一起）是有的，BR/EDR和BLE的TX/RX 中都是通过HitTarget去过滤，即PARAM_INDEX 11；
            hit address，由 hittarget 去产生 4 个 byte 的 sync word  

    3.    WhiteningCoeffValue 的参数在设置Maximum(Enable) Delta F1    Delta F2(Disable)不一样，这个有什么特别的要求吗？
    RTK==》Whitening enable 效果是将数据重新打散,让它能量均匀,等于是编过码的；
           Whitening disable ,就是不打散,原始呈现数据；
           具体参数是PARAM_INDEX 6，一般0x7F Enable Whitenin，0x80 disable Whitenin；

    4.    采用default table测试，在bt_mp_SetParam设定中有些PARAM_INDEX参数没有设定，有哪些PARAM_INDEX参数需要在软件中写死？
     

    RTK==》具体参照您要测试的项目，比如测试4.2 Continue TX Mode中step1：bt_mp_SetParam有说明需要填写的index；

                     每个测试项目中都有说明的；

    5.    GFSK/DQPSK/DPSK

    举个例子： 

    DH5，2DH5，3DH5

    前面的1,2,3对应的是1M，2M，3M，后面的5是指此packet占5个slots； 

    分别对应的调制技术是 

    For GFSK                     DH5 

    For π/4 DQPSK              2DH5 

    For 8DPSK                     3DH5 

    都是用于Payload部分 

    即客户提到的“GFSK/DQPSK/DPSK”分别是1M，2M，3M 

    BLE是单独的，听淼哥说类似于1M；

     

    6. Continue TX：

    当需要看TX power时，请使用Continue TX 测试；

    比如：

    rtlbtmp

    enable uart:/dev/ttyAS1

    bt_mp_Exec 5

    bt_mp_Exec 6

    bt_mp_SetParam  1,0;2,0;3,0;6,0x7F;7,6;9,0x0;11,0x000000c6967e;

    bt_mp_Exec  15           //此时频谱分析仪上可以看到波形；

    bt_mp_Exec  17           //停止发包；

     

    packet TX：

    当需要看具体包所含有的信息的话，请使用packet TX：

    具体SetParam哪些index, 请根据介绍packet TX mode 章节 里表格描述；

    如果要一直打Packet TX：

    bt_mp_HciCmd 0xfce0,26,0x00,0x00,0x00,0xc6,0x96,0x7e,7,0x01,6,0x01,0x7f,0,0,0x01,0x01,0x01,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0x00,0xD8, 0x00

     

     

    代替bt_mp_SetParam

    //0x0000009e8b33

            pData[0] = (HIT_ADDRESS_SET&0xff);                                                   //bt addr 6 bytes

            pData[1] = ((HIT_ADDRESS_SET>>(BYTE_SHIFT*1))&0xff);                   //bt addr

            pData[2] = ((HIT_ADDRESS_SET>>(BYTE_SHIFT*2))&0xff);                   //bt addr

            pData[3] = ((HIT_ADDRESS_SET>>(BYTE_SHIFT*3))&0xff);                   //bt addr

            pData[4] = ((HIT_ADDRESS_SET>>(BYTE_SHIFT*4))&0xff);                   //bt addr

            pData[5] = ((HIT_ADDRESS_SET>>(BYTE_SHIFT*5))&0xff);                   //bt addr

     

            pData[6] = BT_PAYLOAD_TYPE_PRBS9;         //payload_pattern

            pData[7] = 0x01;                                                     //tx_interval

            pData[8] = DEFAULT_TX_POWER_INDEX;              //tx_power_index

            pData[9] = 0x01;                                                     //is_whiten

            pData[10] = WHITENCOEFF_SET;                           //whiten_coeff

            pData[11] = Channel;                                              //rf_channel

            pData[12] = PacketType;                                        //packet_type

            pData[13] = 0x01;                                                   //multirx_enable

            pData[14] = 0x01;                                                   //is_pseudo_outer_mode

            pData[15] = 0x01;                                                   //is_pkt_tx

           

            pData[16] = (PKTHEADER_SET&0xff);;                                                   //payload_header 4 bytes

            pData[17] = ((PKTHEADER_SET>>(BYTE_SHIFT*1))&0xff);                    //payload_header

            pData[18] = ((PKTHEADER_SET>>(BYTE_SHIFT*2))&0xff);                    //payload_header

            pData[19] = ((PKTHEADER_SET>>(BYTE_SHIFT*3))&0xff);                    //payload_header

     

            pData[20] = (TEST_PKT_CNT&0xff);                                                       //tx_packet_count 4 bytes

            pData[21] = ((TEST_PKT_CNT>>(BYTE_SHIFT*1))&0xff);               //tx_packet_count

            pData[22] = ((TEST_PKT_CNT>>(BYTE_SHIFT*2))&0xff);               //tx_packet_count

            pData[23] = ((TEST_PKT_CNT>>(BYTE_SHIFT*3))&0xff);               //tx_packet_count

     

            pData[24] = (PAYLOAD_LENGTH[PacketType]&0xff);    //payload_length 2 bytes

            pData[25] = ((PAYLOAD_LENGTH[PacketType]>>BYTE_SHIFT)&0xff); //payload_length

           

            if(pBtDevice->SendHciCommandWithEvent(pBtDevice, HCI_VENDOR_MP_PACKET_TX_ENABLE_CONFIG, LEN_26_BYTE, pData, 0x0E, pEvtBuf))

                    goto error;

    static unsigned int PAYLOAD_LENGTH[BT_PKT_TYPE_NUM] =

    {

            216,1464,2712,       // 1M

            432,2936,5432,       // 2M

            664,4416,8168,       // 3M

    };

    Mp tool本身支持直接下cmd给fw的，格式为btmp_hci_cmd opcode,data_len,data

     

    7.  关于RX 测试的说明：

    7.1PACKET RX mode
     

    请参考MP tool user guide xxxxx.pdf中4.4 Packet RX Mode 

    测试 PACKET RX mode时，仪器端要用PACKET_TX 去打；

     

    以下为参考流程；

    RX端 CMD ： 

    root@polaris-ibt-8723bs:/ # rtlbtmp 

    rtlbtmp

    ::::::::::::::::::::::::::::::::::::::::::::::::: 

    :::::::: Bluetooth MP Test Tool Starting :::::::: 

    enable xxxxx 

    bt_mp_SetParam  1,36;2,8;3,7;6,0x7F;9,0x1234;11,要与TX端HitTarget一致; 

    bt_mp_Exec 18    // PACKET_RX_START 

    bt_mp_Report 3 

    >bt_mp_Report,3,0x00,-96,0x00000000,0x00000000,0x00000000 


    TX端 CMD(此处TX端也是用RTK 方案代替仪器方式，实际测试时，仪器的操作请follow 此处) ： 

    root@polaris-ibt-8723bs:/ # rtlbtmp 

    rtlbtmp 

    ::::::::::::::::::::::::::::::::::::::::::::::::: 

    :::::::: Bluetooth MP Test Tool Starting :::::::: 

    enable xxxxxxxxx 

    bt_mp_Exec 5

    bt_mp_Exec 6 

    bt_mp_SetParam  1,36;2,8;3,7;4,0xFFF;6,0x7F;7,6;9,0x1234;11,要与RX端HitTarget一致;//如果用仪器来打PACKET TX，请依据这里依次设置仪器端的项目； 

    bt_mp_Exec 12  // PACKET_TX_START

     

    过一会，等TX打完了，再RX端 CMD ： 

    bt_mp_Report 3 

    > bt_mp_Report,3,0x00,-64,0x00f507b0,0x000007ae,0x0000568b 

    bt_mp_Exec 20 // PACKET_RX_STOP 



    7.2 LE RX 

    ==》请参考MP tool user guide xxxxx.pdf中4.7 LE DUT Test Mode (TX/RX)章节有详细描述：

    To start LE RX DUT test mode, follow the steps below: 

    Step 1: bt_mp_SetParam 

    bt_mp_SetParam  1,0;//仪器端也要打相应的channel； 

    Step 2: bt_mp_Exec(LE_RX_DUT_TEST_CMD) 

    bt_mp_Exec 23 

    Step 3：仪器端打TX 包： 

    请把仪器端打tx 的 number 值设成 2 的多少次方，且要 > 1024以上，如 1024 , 1024+512 , 原因是我们 report 机制,buffer 满了,才会上报； 

    等仪器端TX结束后， 

    Step 4: bt_mp_Exec(LE_DUT_TEST_END_CMD) to stop LE RX DUT mode. 

    bt_mp_Exec 24 

    Step 5: bt_mp_Report 11
